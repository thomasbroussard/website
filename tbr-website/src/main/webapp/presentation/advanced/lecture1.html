Now, let's start : Design patterns and Java

Design patterns : What?

  • Design patterns are reusable architectures, bringing all-made solutions to a given problem
  • Using a design pattern must be studied : misapplication can cause severe productivity problems

Design patterns : Why?

  • The usage of design patterns is not always needed, but...
  • It can slightly make the difference, especially when you work with other people
  • Indeed, the vocabulary is precise, and standard (it helps other to know directly what you tried to do)
  • These patterns are very used in the software companies, so knowing them can help you to enter in a complex architecture
  • These patterns can apply in many other languages

Design patterns : Categories

There are several kind of patterns, each of them answering to a specific need
  • Creational patterns help you to handle the creation of objects
  • Structural patterns help you to handle to increase the abilities of objects in a sustainable way
  • Behaviour patterns help you to optimize communications and cross usage of objects
  • Concurrency patterns help you to optimize the way concurrency is managed in your program

Creational patterns

These patterns have in common the creation of objects, first example The Factory Pattern

Object Pool: Definition

The object pool pattern helps you to realize the initialization of a configured amount of objects, which are particularly heavy to set up.
It can be for example, a connection to a DataBase

The Strategy Pattern

The strategy pattern helps you to update the behaviour of your program, under certain condition
Example
  • You try to access to a remote database which does not respond
  • If you don't have a fall back, you cannot record the user-input
  • Solution, change your record strategy to handle the database failure

Exercise
